SkyHelper API

Authentication

The SkyHelper API uses their own API keys to authenticate requests.
You can either use the key query parameter by adding a ?key=token to the end of API requests or using an authorization header by adding a Authorization header to your API requests, where the value of the header is your API token.

Responses

All endpoints in the API return all their responses in JSON, all requests will return a status key which should match the response status code that was returned and a data key for successful responses. A reason key will be returned for failed requests.

Status Code Reason
200 Successful request
400 The request is missing an authentication method (valid key query parameter or an Authentication header)
403 The provided token does not exist
404 There is no player with the given UUID or name or the player has no SkyBlock profiles
429 The Hypixel API rate-limit was reached (The API will return RateLimit-Remaining and RateLimit-Reset headers)
500 There was an error in the SkyHelper API or the Hypixel API returned an unknown error code. Please report these errors back on GitHub
502 Hypixels API is experiencing some technical issues or is unavailable
503 Hypixels API is in maintenance mode
504 Hypixels API returned a Gateway Time-out error

Endpoints

POST /v2/networth

Field Type Description
profileData Object The profile player data from the Hypixel API (profile.members[uuid])
bankBalance Number The player's bank balance from the Hypixel API (profile.banking?.balance)
onlyNetworth Boolean (default: false) If true, only the networth will be returned

POST/v2/networth/item

Field Type Description
itemData Object The parsed item data of an item from the profiles endpoint

GET /v2/profiles/:user

GET /v2/profile/:user/:profile

GET /v1/profiles/:user

GET /v1/profile/:user/:profile

GET /v1/items/:user

GET /v1/items/:user/:profile

GET /v1/fetchur

Parameter Description
user This can be the UUID of a user or the name
profile This can be the users profile id or name
